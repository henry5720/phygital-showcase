# AR V3 Interactive Demo 完整對齊頁設計文件

**日期**: 2026-06-06  
**狀態**: 已確認，待實作計畫

## 1. 目標

建立 `/ar/v3` 路由，以 MindAR 官方 `advanced.html` (Interactive Example) 為藍本，在 React 專案中 1:1 還原其互動流程與視覺體驗。

主要包含：
- 完整的進場動畫時序 (Avatar -> Portfolio -> Info Icons)
- 左右切換的 Portfolio Panel
- 點擊 Preview 切換影片播放
- 四顆 Info Icons (profile/web/email/location) 與對應文字區
- 自定義的 Scanning Overlay (掃描框動畫)
- 使用官方 `softmind` 素材以確保行為一致性

## 2. 非目標

- 不更動或刪除 `/ar/v2` 驗證頁
- 不替換成品牌自有素材 (第一階段保留官方素材)
- 不進行跨元件的 AR 抽象化，保持 v3 邏輯獨立

## 3. 路由策略

- 新增 `/ar/v3`
- 定位為「Interactive Demo 對齊版」

## 4. 技術架構

採用與 v2 相似但更完整的拆層：

- `ArV3Page`: 頁面殼層，處理 React Router。
- `ArV3Scene`: A-Frame `<a-scene>` 與素材宣告。
- `ar-v3.ts`: 封裝官方 `advanced.html` 的指令式互動邏輯 (Imperative logic)，包含 `setInterval` 動畫、事件監聽與資源清理。

## 5. 互動細節對齊 (1:1 原則)

### A. 進場時序
1. `targetFound` 觸發。
2. `showAvatar`: 調整 avatar Z 軸動畫。
3. `showPortfolio`: 調整 panel Y 軸動畫，結束後顯示左右按鈕。
4. `showInfo`: 四顆 icon 依序 (delay 300ms) 顯示。

### B. Portfolio 互動
- 左右按鈕切換 Item 0/1/2。
- Item 0 包含 Video Preview 圖。
- 點擊 Preview 圖：隱藏圖片，根據瀏覽器支援度播放 webm 或 mp4 影片。

### C. Info 互動
- 點擊 Icon：下方文字區更新對應文案。
- 點擊 Web 連結文案：跳轉至外部連結。

### D. 掃描體驗
- 隱藏 A-Frame 預設 UI。
- 使用 HTML/CSS 實作與官方一致的自定義掃描框。

## 6. 素材策略

所有素材放在 `public/assets/ar-v3/`，確保與 v2 隔離。
- `targets/card.mind`
- `models/softmind/scene.gltf`
- `icons/` (web, location, profile, phone, email, play, left, right)
- `portfolio/` (preview images, videos)

1. `/ar/v3` 可進入且不報錯。
2. 掃描成功後，進場動畫與官方 `advanced.html` 一致。
3. 所有互動功能 (Portfolio 切換、影片播放、Icon 點擊文字更新) 正常運作。
4. 掃描框視覺與官方一致。
5. 離開頁面後相機與 Overlay 徹底清理。
