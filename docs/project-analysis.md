# 植酌 Fizz't Phygital Showcase — 專案分析與發展建議

> 基於 PM 提案（`docs/fizzt/02.png`）與當前實作進度的對照分析
> 日期：2026-06-06

---

## 一、專案定位

**目標**：打造一個 QR Code 驅動的品牌體驗閉環  
**核心命題**：掃描 → 體驗 → 了解產品 → 加入會員 → 持續經營

PM 提案的完整意圖不只是一次性的行銷活動頁，而是一個**可追蹤、可分眾、可導購的品牌體驗轉換系統**。  
目前實作完成了「體驗端」的功能骨架，尚未完整閉環。

---

### 已對齊 ✅

| PM 提案功能點 | 目前實作 | 備註 |

### 部分完成 ⚠️

| PM 提案功能點 | 目前狀態 | 說明 |

   │ 3D 瓶身浮現   │   │ 分眾標籤綁定      │
   │ 品牌故事影片  │   │ 推薦產品連結      │
   │ 五感互動      │   └──────────────────┘

           │
           ▼
   ┌─ Product Page ──┐
   │ ScrollTrigger   │
   │ 品牌故事捲動    │
   │ 產品特色        │
   │ CTA 加入 LINE   │
   └─────────────────┘
           │
           ▼
   ┌─ LINE CRM ───────────────────┐

   └──────────────────────────────┘

### 目前實際旅程

```
Landing (文字+按鈕)
   │
   ├─ WebAR (掃描 → 熱點 → 影片) ── 看完就結束，無下一步
   ├─ Quiz (5題 → 結果 → LINE)
   └─ LINE 直接跳 URL ── 無參數
```

**關鍵斷點**：WebAR 走完後沒有明確的轉換步驟，Product Page 不存在，LINE 入口無法識別使用者來源。

## 四、建議發展優先級

### P0 — 補齊核心體驗閉環

| 項目 | 工時估計 | 說明 |
|------|----------|------|
| WebAR 完成 CTA | 1-2 天 | 影片播完後顯示懸浮按鈕「加入 LINE 領取調飲配方」，帶 `?source=ar` 參數 |
| Product Page 實作 | 3-5 天 | GSAP ScrollTrigger 捲動敘事頁，承接 AR「了解更多」流量 |
| Quiz 結果頁 LINE 帶參數 | 半天 | `?source=quiz&result={type}`，為未來 CRM 分眾鋪路 |

### P1 — 提升品牌體驗質感

| 項目 | 說明 |
|------|------|
| Landing 加入瓶身主視覺 + 情境圖 | 目前無 hero image（`hero.png` 不存在） |
| WebAR 敘事流程化 | 三個熱點改為階段式引導（先播故事 → 再出現下一個） |
| 五感互動 overlay | 提案亮點，AR 中顯示感官 icon + 對應文案切換 |

### P2 — 數據與 CRM 基礎

| 項目 | 說明 |
|------|------|
| GA4 gtag 埋點 | 5 個事件已定義（見 design spec），引入 gtag script 即可 |
| LINE LIFF 或 Deep Link | 相比純 URL 跳轉，LIFF 可帶入更多使用者資訊 |
| 導購問卷 | 直接加入 LINE 後，可用問卷蒐集偏好進行分眾 |

## 五、風險提醒

| 風險 | 說明 |
|------|------|
| **WebAR 轉換率** | 目前 AR 體驗結束後無明確行動呼籲，使用者看完影片就離開，損失轉換機會 |
| **素材缺失** | `hero.png` 不存在於 `public/assets/`，Landing 無法顯示產品主視覺 |
| **CRM 斷層** | LINE 入口無參數，未來即使接 CRM 也無法回溯使用者來源與行為 |
| **Product Page 空白** | `/product` 路由已存在但無內容，若專案對外展示時被點到會呈現錯誤印象 |
| **測驗題目固定** | 5 題寫死於 config，若要 A/B 測試或優化需手動更新 |
